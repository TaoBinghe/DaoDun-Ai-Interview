```
【主流程逻辑（简）】

  用户 ──► 应用层发起面试
              │
              ▼
  服务层创建会话、拉取岗位/题库/简历上下文
              │
              ├──► 模型层：LLM 多轮对话（可选 RAG 注入知识片段）
              │
              ├──► 语音场景：WS 上传音频 → STT → 同会话编排；TTS 回播音频/字幕
              │
              ├──► 情绪：前端帧 → 后端转发 → 情绪服务 → 时间线落会话
              │
              ▼
  结束面试 ──► 异步生成评估报告 ──► 写入 evaluation_report
              │
              ▼
  应用层报告页轮询/拉取 GET /sessions/{id}/evaluation ──► 雷达图与文字报告展示


【论坛逻辑】

  用户 ──► 应用层打开论坛列表 / 帖子详情
              │
              ▼
  服务层：GET/POST 帖子与评论等接口；JWT 识别作者，内容校验与频控
              │
              ▼
  数据层：帖子与评论持久化，作者关联 users；不经过模型层亦可独立完成
```

## 与参考图的分层对应说明

| 参考图层次 | 本项目对应 |
|-----------|-----------|
| 应用层 | `frontend/` 各业务视图与布局；含 **交流论坛** |
| 服务层 | `backend/` Spring Boot、REST、`/ws/interview`、JWT、异步评估；**论坛 REST 与面试 API 同级** |
| 模型层 | 方舟 LLM、RAG+向量检索、火山语音、情绪分析服务 |
| 数据层 | PostgreSQL（含向量）、Redis、QAdocs 知识源；**含论坛业务表（帖子、评论等）** |
