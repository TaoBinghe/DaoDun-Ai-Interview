

# 需求说明书：用户注册与登录模块 (Epic: User Authentication)

## 1. 概述

本模块旨在为“AI模拟面试与能力提升软件”提供安全、便捷的用户身份认证服务。作为系统的入口，该模块需保证用户数据的安全性，并提供流畅的交互体验，确保学生能够快速建立个人档案并保留面试记录。

## 2. 功能需求详细说明

### 2.1 用户注册 (User Registration)

**业务目标**：允许新用户创建平台账号。

- **输入字段与校验规则**：
  - **邮箱 (Email)**：
    - 规则：必须符合标准电子邮箱格式（如 `^[A-Za-z0-9+_.-]+@(.+)$`）。
    - 校验：必须全局唯一。若邮箱已被注册，提示“该邮箱已注册，请直接登录”。
  - **验证码 (Verification Code)**：
    - 规则：6位纯数字。
    - 有效性：发送后 5 分钟内有效。
    - 防刷机制：同一邮箱每 60 秒内只能请求发送一次验证码；
  - **用户名 (Username)**：
    - 规则：长度为 5-10 个字符。支持中英文、数字组合，但总长度必须在此范围内。
    - 校验：必须全局唯一。若用户名已存在，提示“该用户名已被占用，请尝试其他名称”。
  - **密码 (Password)**：
    - 规则：长度为 5-16 位。
    - 复杂度：**必须**包含英文大小写字母和数字的组合，不允许纯数字或纯字母。
- **业务流程**：
  1. 用户输入邮箱地址。
  2. 用户点击“获取验证码”，前端倒计时 60 秒。
  3. 后端生成 6 位验证码，存入 Redis 并设置 5 分钟过期时间，通过邮件服务发送至用户邮箱。
  4. 用户填写验证码、用户名、密码。
  5. 点击“注册”，后端进行各项字段的格式和唯一性校验。
  6. 校验通过后，对密码进行加密（如 BCrypt）并存入数据库，返回注册成功信息，并可选择直接自动登录。

### 2.2 用户登录 (User Login)

**业务目标**：允许已注册用户安全进入系统，恢复其历史面试记录和个性化配置。

- **登录方式一：账号密码登录**
  - 输入：用户名、密码。
  - 校验：比对数据库中的用户名及加密后的密码。若错误，提示“用户名或密码错误”（出于安全考虑，不具体指明是哪个错误）。
- **登录方式二：邮箱验证码登录**
  - 输入：已注册的邮箱、验证码。
  - 流程：输入邮箱 -> 获取验证码 -> 输入验证码 -> 登录。
  - 校验：验证邮箱是否存在（不存在则提示“该邮箱未注册”），验证码是否正确及过期。
- **记住我功能 (Remember Me)**
  - 触发：用户在登录界面勾选“记住我”。
  - 机制：采用 Token 机制（如 JWT）。
    - 不勾选“记住我”：Token 有效期较短（如 2 小时）或仅在浏览器 Session 期间有效。
    - 勾选“记住我”：发放一个长期有效的 Refresh Token（如 7 天或 14 天）。用户关闭浏览器重新打开时，前端自动携带 Refresh Token 换取新的 Access Token 从而实现免密登录。

## 3. 非功能性需求 (Non-Functional Requirements)

### 3.1 安全性 (Security)

- **密码存储**：数据库中绝对禁止明文存储密码。必须使用带 Salt 的哈希算法（推荐使用 Spring Security 自带的 `BCryptPasswordEncoder`）。
- **接口安全**：所有注册和登录接口需进行防暴力破解限制。注册获取验证码的接口需防止被恶意调用刷邮件额度。
- **Token 安全**：JWT 需设置合理的过期时间，且 Payload 中不应包含用户密码等敏感信息，仅包含用户 ID 和基础权限标识。

### 3.2 性能 (Performance)

- 邮件发送接口的响应时间应在 2 秒以内（建议后端采用异步线程池处理邮件发送，避免阻塞主线程）。
- 登录验证接口响应时间应小于 500 毫秒。

## 4. 前后端对接接口规划建议 (API Design)

为方便前后端分离开发，建议规划以下 RESTful 接口：

| **接口名称**       | **请求方法** | **路径**                       | **核心参数 (JSON)**                     |
| ------------------ | ------------ | ------------------------------ | --------------------------------------- |
| **发送注册验证码** | POST         | `/api/auth/send-register-code` | `email`                                 |
| **发送登录验证码** | POST         | `/api/auth/send-login-code`    | `email`                                 |
| **用户注册**       | POST         | `/api/auth/register`           | `email`, `code`, `username`, `password` |
| **密码登录**       | POST         | `/api/auth/login/password`     | `username`, `password`, `rememberMe`    |
| **验证码登录**     | POST         | `/api/auth/login/email`        | `email`, `code`, `rememberMe`           |

## 5. 数据库表设计参考 (User Table)

| **字段名**    | **数据类型** | **约束**                    | **说明**                  |
| ------------- | ------------ | --------------------------- | ------------------------- |
| `id`          | BIGINT       | PK, Auto Increment          | 主键 ID                   |
| `username`    | VARCHAR(20)  | UNIQUE, NOT NULL            | 用户名 (限制5-10位)       |
| `email`       | VARCHAR(50)  | UNIQUE, NOT NULL            | 用户邮箱                  |
| `password`    | VARCHAR(255) | NOT NULL                    | 加密后的密码              |
| `status`      | TINYINT      | NOT NULL, DEFAULT 1         | 状态（1正常，0锁定/禁用） |
| `create_time` | DATETIME     | DEFAULT CURRENT_TIMESTAMP   | 注册时间                  |
| `update_time` | DATETIME     | ON UPDATE CURRENT_TIMESTAMP | 更新时间                  |
