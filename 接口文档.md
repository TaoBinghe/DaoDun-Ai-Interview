# AI模拟面试与能力提升平台 - 接口文档 V1.0

## 1. 用户认证相关接口

### 1.1 发送注册验证码

#### 1.1.1 基本信息

> 请求路径：/api/auth/send-register-code
>
> 请求方式：POST
>
> 接口描述：向指定邮箱发送注册用验证码，验证码 5 分钟内有效；同一邮箱 60 秒内只能请求一次；若邮箱已注册则提示直接登录。

#### 1.1.2 请求参数

请求参数格式：application/json

请求参数说明：

| 参数名称 | 说明 | 类型   | 是否必须 | 备注                     |
| -------- | ---- | ------ | -------- | ------------------------ |
| email    | 邮箱 | string | 是       | 符合标准电子邮箱格式     |

请求数据样例：

```json
{
    "email": "user@example.com"
}
```

#### 1.1.3 响应数据

响应数据类型：application/json

响应参数说明：

| 名称 | 类型   | 是否必须 | 默认值 | 备注                         | 其他信息 |
| ---- | ------ | -------- | ------ | ---------------------------- | -------- |
| code | number | 必须     |        | 响应码，200-成功，非200-失败 |          |
| msg  | string | 非必须   |        | 提示信息                     |          |
| data | object | 非必须   |        | 返回的数据，本接口为 null    |          |

响应数据样例：

```json
{
    "code": 200,
    "msg": "验证码已发送，请查收邮件",
    "data": null
}
```

#### 1.1.4 备注说明

> 若该邮箱已注册，接口返回 code 为 400，msg 为「该邮箱已注册，请直接登录」。
> 若 60 秒内重复请求，返回「发送过于频繁，请 60 秒后再试」。

### 1.3 用户注册

#### 1.3.1 基本信息

> 请求路径：/api/auth/register
>
> 请求方式：POST
>
> 接口描述：使用邮箱、验证码、用户名、密码完成注册；密码使用 BCrypt 加密存储；校验验证码、邮箱与用户名的唯一性。

#### 1.3.2 请求参数

请求参数格式：application/json

请求参数说明：

| 参数名称 | 说明     | 类型   | 是否必须 | 备注                                           |
| -------- | -------- | ------ | -------- | ---------------------------------------------- |
| email    | 邮箱     | string | 是       | 符合标准电子邮箱格式，且未被注册               |
| code     | 验证码   | string | 是       | 6 位纯数字，须为发送注册验证码接口下发的验证码 |
| username | 用户名   | string | 是       | 5~10 个字符，支持中英文、数字，全局唯一        |
| password | 密码     | string | 是       | 5~16 位，须包含英文大小写字母和数字的组合     |

请求数据样例：

```json
{
    "email": "user@example.com",
    "code": "123456",
    "username": "张三abc",
    "password": "Abc12345"
}
```

#### 1.3.3 响应数据

响应数据类型：application/json

响应参数说明：

| 名称 | 类型   | 是否必须 | 默认值 | 备注                         | 其他信息 |
| ---- | ------ | -------- | ------ | ---------------------------- | -------- |
| code | number | 必须     |        | 响应码，200-成功，非200-失败 |          |
| msg  | string | 非必须   |        | 提示信息                     |          |
| data | object | 非必须   |        | 返回的数据，本接口为 null    |          |

响应数据样例：

```json
{
    "code": 200,
    "msg": "注册成功",
    "data": null
}
```

#### 1.3.4 备注说明

> 验证码错误或过期时返回「验证码已过期，请重新获取」或「验证码错误」。
> 用户名已存在时返回「该用户名已被占用，请尝试其他名称」。
> 邮箱已注册时返回「该邮箱已注册，请直接登录」。



### 1.4 账号密码登录

#### 1.4.1 基本信息

> 请求路径：/api/auth/login/password
>
> 请求方式：POST
>
> 接口描述：使用用户名和密码登录；成功后返回 Access Token、Refresh Token 及用户基本信息；rememberMe 为 true 时 Refresh Token 有效期为 14 天，否则为 7 天。

#### 1.4.2 请求参数

请求参数格式：application/json

请求参数说明：

| 参数名称   | 说明     | 类型    | 是否必须 | 备注                                       |
| ---------- | -------- | ------- | -------- | ------------------------------------------ |
| username   | 用户名   | string  | 是       | 已注册的用户名                             |
| password   | 密码     | string  | 是       | 明文密码                                   |
| rememberMe | 记住我   | boolean | 否       | 默认 false；为 true 时延长 Refresh Token 有效期 |

请求数据样例：

```json
{
    "username": "张三abc",
    "password": "Abc12345",
    "rememberMe": false
}
```

#### 1.4.3 响应数据

响应数据类型：application/json

响应参数说明：

| 名称   | 类型   | 是否必须 | 默认值 | 备注                         | 其他信息 |
| ------ | ------ | -------- | ------ | ---------------------------- | -------- |
| code   | number | 必须     |        | 响应码，200-成功，非200-失败 |          |
| msg    | string | 非必须   |        | 提示信息                     |          |
| data   | object | 必须     |        | 登录结果                     |          |
| \|-accessToken  | string | 非必须 | | 访问令牌，用于请求头 Authorization | |
| \|-refreshToken | string | 非必须 | | 刷新令牌，用于换取新的 accessToken | |
| \|-userId      | number | 非必须 | | 用户主键 ID                  |          |
| \|-username    | string | 非必须 | | 用户名                       |          |
| \|-email       | string | 非必须 | | 邮箱                         |          |

响应数据样例：

```json
{
    "code": 200,
    "msg": "登录成功",
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "userId": 1,
        "username": "张三abc",
        "email": "user@example.com"
    }
}
```

#### 1.4.4 备注说明

> 用户名或密码错误时，统一返回 code 400，msg「用户名或密码错误」（不区分具体错误项）。
> 登录成功后，后续请求需在请求头中携带：Authorization: Bearer \<accessToken\>。Access Token 有效期 2 小时；过期后可使用 refreshToken 换取新的 accessToken（刷新接口若已实现则在此补充路径）。

## 2. 通用说明

### 2.1 基础路径

> 如无特殊说明，接口基础路径为：`http://localhost:8080`（以实际部署为准）。

### 2.2 统一响应结构

| 名称 | 类型   | 说明                         |
| ---- | ------ | ---------------------------- |
| code | number | 200 表示成功，400/401/403/500 等表示失败 |
| msg  | string | 提示信息                     |
| data | object | 业务数据，失败时多为 null    |

### 2.3 认证方式

> 需要登录的接口，请在请求头中携带：  
> **Authorization: Bearer \<accessToken\>**  
> 其中 accessToken 为登录接口返回的 data.accessToken。

### 2.4 错误码说明

| code | 说明                     |
| ---- | ------------------------ |
| 200  | 成功                     |
| 400  | 请求参数错误或业务校验失败 |
| 401  | 未登录或 Token 无效/过期  |
| 403  | 无权限（如账号被禁用）    |
| 500  | 服务器内部错误            |
